1. Structures and Classes share the same method of writng. Both contains namespace and main methods. but keyword 
used is Struct and Class respectively.

2. Class is reference type whereas struct is a value type

3. Memory allocation for the instance of the class is done in managed heaps while in struct it is in stacks

4. We use class for representing an entity with larger volumes of data whereas struct is used for representing 
entity with smaller volumes of data.

5. All the predefined data types in the libraries which are of reference types are defined using class
Whereas those of value type are defined using struct keywords. 
ex string , object are ref types
ex int bool are value types. 
simply got to deffinitions of above data types and see how they are made.

6. In classes we have to use the new keyword to create instance and access its methods. But in struct
Using new keyword followed by the constructor is optional. Refer the CS file of structures

7. Remember if you dont init struct using new then its variables shoud be separately initialized by selecting its instance and 
instance.var_name = value.

8. Under a class we can define any constructor that is parameterized or parameterless. If no constructor is defined then it will 
be implicit constructor which will be default. In case of structure parameterless constructor is always default and cant be
defined again. We can define another parameterized constructor in Structure. If we define parameterless constructor . Build will fail

9. In class if we define n number constructors . After compilation total constructors will be n. There wont be implicit
If in a class there are 0 constructors after compilation total 1 constructors will be present

But in a struct if n constructors are defined. After compilation total constructors will be n+1. Plus 1 is implicit.

10 Structures dont support inheritance. It cant inhreit from class or structure or No class can inherit them as well.
